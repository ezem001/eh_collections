# Postman Native Git layout

У цьому репозиторії правильна структура для Postman Native Git тепер така:

- `workspaces/Demo`
- `workspaces/Preprod`
- `workspaces/Stage`

Кожна з цих папок має бути підключена в Postman окремо до однойменного workspace. Це прибирає конфлікти, коли в різних workspace є колекції з однаковими назвами.

Поточна папка `postman/` у корені репозиторію містить змішаний старий flat-layout і використовується тільки як тимчасове джерело для ручної міграції. Її краще більше не використовувати для нових pull/push.

## Як мігрувати

1. У Postman відкрийте workspace `Demo` і підключіть папку `workspaces/Demo`.
2. Повторіть окремо для `Preprod` і `Stage`.
3. Для кожного workspace виконайте `Pull from Postman Cloud` вже в його власну папку.
4. Після перевірки нової структури старий змішаний кореневий `postman/` можна буде прибрати окремим кроком.

## Чому так правильно

- Postman Native Git працює стабільніше за схемою `один локальний root = один Postman workspace`.
- Однакові назви колекцій між `Demo`, `Preprod`, `Stage` більше не конфліктують, бо лежать у різних директоріях.
- Це природно лягає в GitHub: кожен workspace зберігається як окремий набір файлів.
